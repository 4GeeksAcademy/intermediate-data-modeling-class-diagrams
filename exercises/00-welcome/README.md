## What you will do

You will practice **intermediate data modeling** with **UML class diagrams** for an **online course platform**. Each step adds concepts to the same diagram in [diagram.4geeks.com](https://diagram.4geeks.com/).

## diagram.4geeks.com: what to expect

The editor focuses on:

- **Classes** with **typed attributes** (`int`, `string`, `boolean`, `date`)
- **Relationships** between classes
- **Text labels** on links for cardinality (`1`, `N`, `*`)

It does **not** require full UML notation. In this practice:

- Use **labels on connectors** instead of special arrowheads for cardinality.
- Use **`string` properties** for statuses (for example `status: string` with values `active`, `completed` described in the step text) instead of UML enumerations.
- Model **many-to-many** with a visible class such as `Enrollment` or `Review`, not a nameless link.

If the UI lacks a UML feature, a **plain text label** on the relationship or a **string field** on a class is enough.
