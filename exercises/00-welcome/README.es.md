# Bienvenida: Como funciona esta practica

Version en ingles: [README.md](./README.md)

## Que vas a hacer

Practicaras **modelado de datos intermedio** con **diagramas de clases UML** para una **plataforma de cursos online**. Cada paso agrega conceptos al mismo diagrama en [diagram.4geeks.com](https://diagram.4geeks.com/).

## diagram.4geeks.com: que esperar

El editor se centra en:

- **Clases** con **atributos tipados** (`int`, `string`, `boolean`, `date`)
- **Relaciones** entre clases
- **Etiquetas de texto** en los enlaces para cardinalidad (`1`, `N`, `*`)

No exige notacion UML completa. En esta practica:

- Usa **etiquetas en los conectores** en lugar de flechas UML especiales para cardinalidad.
- Usa **propiedades `string`** para estados (por ejemplo `status: string` con valores `active`, `completed` descritos en el paso) en lugar de enumeraciones UML.
- Modela **muchos a muchos** con una clase visible como `Enrollment` o `Review`, no un enlace sin nombre.

Si la interfaz no tiene una funcion UML, basta una **etiqueta de texto** en la relacion o un **campo string** en la clase.
